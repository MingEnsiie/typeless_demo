§01项目背景

为什么做这个

Typeless 是一款 AI 语音输入工具，用 LLM 把口语转写"洗"成可发送的书面文本。本项目目标是用单一网页 demo 复刻其交互与体验，所有 UI 与功能跑在浏览器里，pnpm dev 即开即用。

🎯 目标

单 Web SPA · 不需要后端 · 不需要桌面壳。

复刻 typeless 全部核心功能：听写 / 润色 / 上下文 / 框选改写 / 语音翻译 / 个性化 / 模型管理。

🧠 模型策略

ASR：Groq Whisper Turbo 默认 · 浏览器 transformers.js 选配。

LLM：DeepSeek V4 Flash 默认 · MiMo / Ollama 切换。

🔐 边界与妥协

浏览器无法注入到其他应用 → 改为页面内"模拟目标应用"+ 系统剪贴板。

API Key 暴露于 localStorage → UI 显眼提示 demo 用途。

⚙️ CORS 方案

Vite dev server proxy 转发 /api/llm/* 与 /api/asr/*，零后端代码。

v0.2 变更： 补两条核心场景 — ① D 类（框选改写） 扩展为 textarea 内浮动浮层菜单 + 6 预设动作 + 自定义语音指令。② 新增 J 类（语音翻译） 独立模式：专属热键 Cmd+Shift+T、顶部目标语言下拉、原文/译文双列展示。视觉重做为浅背景蓝色科技风。

§03核心数据流

主路径：按下热键 → 文本落入演示区 · 全程 ≤ 3 秒

图 2 · 主路径（听写）端到端数据流。框选改写与语音翻译共用左半 ASR 链路，在 polishing 节点切换 prompt → 见 §09。
① 采集

用户按住空格触发 HOTKEY_DOWN。useRecorder 拿到 16k mono Float32 流，分发给 VAD worker 与累积缓冲。

② 截断

VAD 检测到 600ms 静音 → VAD_END；或用户松开键 → HOTKEY_UP。任一即进入 transcribing。

③ 转写

PCM→WAV blob multipart POST 到 Groq Whisper（默认）。带 language=auto + 词典提示。

④ 润色 / 改写 / 翻译

LLM 走 OpenAI 兼容 chat completions（DeepSeek 默认），SSE 流式。根据模式切换三套不同的 prompt。

⑤ 注入

写入 DemoSurface textarea 或替换选区，同时 navigator.clipboard.writeText 写系统剪贴板。

⑥ 学习

30s 内若用户手动改动 textarea，diff 存为 passive-learning 样本，下次注入到 prompt。

§04核心状态机

5 个状态 · 3 种 mode 分支 · 每步带超时熔断

图 3 · xstate 状态机：同一套状态骨架，context.mode 决定 polish 节点调用的 prompt（润色 / 改写 / 翻译），保证逻辑可控。
src/machine/typeless.machine.ts
import { setup, assign, fromPromise } from 'xstate';

type Mode = 'dictate' | 'rewrite' | 'translate';

export const typelessMachine = setup({
  types: {} as {
    context: {
      mode: Mode;
      audio: Float32Array | null;
      raw: string; output: string;
      // rewrite 分支专用
      selectionText?: string; selectionRange?: { start: number; end: number };
      rewritePreset?: string;       // 'formal' | 'casual' | 'shorten' | 'expand' | 'fix-typo' | 'translate'
      // translate 分支专用
      targetLang?: string;          // 'en' | 'zh' | 'ja' | 'ko' | ...
      appContext: AppContext;
      error: string | null;
    };
    events:
      | { type: 'HOTKEY_DOWN'; mode?: Mode }
      | { type: 'HOTKEY_UP' }
      | { type: 'VAD_END' }
      | { type: 'REWRITE_PRESET'; preset: string }   // 选区菜单按钮触发，无需录音
      | { type: 'CANCEL' };
  },
  actors: {
    transcribe: fromPromise(async ({ input }) => asr.transcribe(input.pcm)),
    polish:     fromPromise(async ({ input }) => {
      switch (input.mode) {
        case 'rewrite':   return llm.rewrite(input.selection, input.instruction, input.ctx);
        case 'translate': return llm.translate(input.raw, input.targetLang, input.ctx);
        default:           return llm.polish(input.raw, input.ctx);
      }
    }),
    inject:     fromPromise(async ({ input }) =>
      input.mode === 'rewrite'
        ? replaceSelection(input.range!, input.text)
        : appendDemoText(input.text)),
  },
}).createMachine({
  id: 'typeless', initial: 'idle',
  states: {
    idle: { on: {
      HOTKEY_DOWN: { target: 'recording', actions: assign({ mode: ({ event }) => event.mode ?? 'dictate' }) },
      REWRITE_PRESET: { target: 'polishing', actions: assign({ mode: 'rewrite' }) },
    } },
    recording: { on: { HOTKEY_UP: 'transcribing', VAD_END: 'transcribing', CANCEL: 'idle' }, after: { 60000: 'idle' } },
    transcribing: { invoke: { src: 'transcribe', onDone: 'polishing', onError: 'idle' }, after: { 15000: 'idle' } },
    polishing:    { invoke: { src: 'polish',     onDone: 'injecting', onError: 'idle' }, after: { 15000: 'idle' } },
    injecting:    { invoke: { src: 'inject',     onDone: 'idle',      onError: 'idle' } },
  },
});

10 类核心功能 · 与 typeless 对齐 + 2 类强化

标 ★ 为 demo 形态妥协；NEW 标识 v0.2 新增/强化。

A核心听写
  • A1默认热键 Space（push-to-talk）；可改 Cmd/Ctrl+.
  • A2实时波形浮层（32 频段 + 录音时长 + 状态色）

  • B1去口头禅（嗯、啊、那个、um、uh、like）

  • B4自动标点、断句、首字母大写
  • B5列表识别（第一/第二 → 数字列表）

  • C3Markdown 模式开关 + react-markdown 渲染

D框选改写 v0.2 强化
  • D1textarea 内选中文本（任意长度 ≥ 2 字符）→ 浮动菜单 SelectionPopover 跟随选区底部出现

  • D3菜单含 自定义指令：按住麦克风按钮录指令（"改成疑问句"、"加 emoji"）→ ASR → rewrite prompt

  • D5流式替换：SSE token 实时替换选区内容（背景高亮 cyan 提示进行中），完成后取消高亮
  • D6失败回退：LLM 出错时保留原选区文字 + 红色边框提示，不破坏用户内容
  • D7Undo：单次替换走浏览器原生 textarea undo stack（execCommand insertText）

11.2 · LLM 客户端（三模式统一）

src/lib/llm/index.ts
export const llm = {
  async polish(raw: string, ctx: PolishCtx): Promise<string> {
    const sys = renderPolishPrompt(ctx);
    return collect(chatStream([{ role: 'system', content: sys }, { role: 'user', content: raw }], ctx.opts));
  },
  async rewrite(selection: string, instruction: string | RewritePreset, ctx: RewriteCtx): Promise<string> {
    const sys = renderRewritePrompt(ctx);
    const usr = `SELECTION:\n${selection}\n\nINSTRUCTION:\n${typeof instruction === 'string' ? instruction : presetMap[instruction]}`;
    return collect(chatStream([{ role: 'system', content: sys }, { role: 'user', content: usr }], ctx.opts));
  },
  async translate(raw: string, targetLang: string, ctx: TranslateCtx): Promise<string> {
    const sys = renderTranslatePrompt({ ...ctx, targetLang });
    return collect(chatStream([{ role: 'system', content: sys }, { role: 'user', content: raw }], ctx.opts));
  },
};

11.3 · 选区监听 hook NEW

src/hooks/useTextareaSelection.ts
export interface Selection {
  start: number; end: number; text: string;
  rect: DOMRect | null;   // 选区末端的 caret 位置，用于 floating-ui 锚点
}

export function useTextareaSelection(ref: RefObject<HTMLTextAreaElement>): Selection | null {
  const [sel, setSel] = useState<Selection | null>(null);
  useEffect(() => {
    const el = ref.current; if (!el) return;
    const update = () => {
      const { selectionStart, selectionEnd, value } = el;
      if (selectionEnd - selectionStart < 2) return setSel(null);
      const text = value.slice(selectionStart, selectionEnd);
      const rect = caretRectInTextarea(el, selectionEnd);   // 用 mirror div 算坐标
      setSel({ start: selectionStart, end: selectionEnd, text, rect });
    };
    el.addEventListener('select', update);
    el.addEventListener('keyup', update);
    el.addEventListener('mouseup', update);
    return () => {
      el.removeEventListener('select', update);
      el.removeEventListener('keyup', update);
      el.removeEventListener('mouseup', update);
    };
  }, [ref]);
  return sel;
}
关键技巧：textarea 没有原生 API 返回选区的 DOM 坐标。常见解法是克隆 mirror div，用同样字体 / 宽度 / 内边距，把 textarea 的前缀文本塞进去，末端插一个 marker span，getBoundingClientRect() 即为光标位置。社区库 textarea-caret 可直接用。

import { useFloating, autoUpdate, flip, shift, offset } from '@floating-ui/react';

const PRESETS = [
  { id: 'formal',   label: '更正式' },
  { id: 'casual',   label: '更口语' },
  { id: 'shorten',  label: '简化'   },
  { id: 'expand',   label: '扩写'   },
  { id: 'fix-typo', label: '修错'   },
  { id: 'translate',label: '🌐 译'  },
] as const;

export function SelectionPopover({ selection, onPreset, onCustom }: Props) {
  const { refs, floatingStyles } = useFloating({
    placement: 'bottom-start',
    whileElementsMounted: autoUpdate,
    middleware: [offset(8), flip(), shift({ padding: 8 })],
  });
  if (!selection?.rect) return null;
  // 用 selection.rect 作为虚拟 reference
  refs.setReference({ getBoundingClientRect: () => selection.rect! });
  return (
    <div ref={refs.setFloating} style={floatingStyles} className="sel-popover">
      <header>✨ AI 改写选区 <span>{selection.text.length} 字</span></header>
      <div className="presets">
        {PRESETS.map(p => <button key={p.id} onClick={() => onPreset(p.id)}>{p.label}</button>)}
      </div>
      <MicButton onCommit={onCustom} />
    </div>
  );
}

§12Prompt 设计

12.1 · 润色 prompt（dictate 模式）

src/lib/llm/prompts/polish.v1.ts
You are Typeless, an AI voice dictation assistant. Polish the user's raw
speech-to-text transcript into clean written text following these rules:

1. REMOVE filler words (um, uh, like, 嗯, 啊, 那个, 就是, 然后, repeated stutters).
2. RESOLVE self-corrections: keep only the user's final intent.
3. ADD punctuation, capitalization, paragraph breaks where natural.
4. PRESERVE the user's language. Mixed languages stay mixed. Do NOT translate.
5. PRESERVE meaning. Do NOT add/omit/rephrase ideas. Do NOT answer.
6. If user dictates a list ("first... second..."), format as numbered list.
7. Adapt tone to target app: {{tone}} ({{formality}}).
8. User dictionary (prefer these spellings): {{dictionary_terms}}.
9. {{#if markdown}}Output Markdown.{{else}}Output plain text.{{/if}}
10. Output ONLY the polished text. No preamble.

12.2 · 改写 prompt（rewrite 模式）NEW

src/lib/llm/prompts/rewrite.v1.ts
You are an AI text editor. The user has selected a piece of text and wants
you to rewrite it according to their instruction.

Rules:
1. Output ONLY the rewritten text. No preamble, no quotes, no commentary.
2. PRESERVE the user's language (do not translate unless instruction says so).
3. Match the surrounding context tone: {{app_context.tone}} ({{formality}}).
4. Keep proper nouns and dictionary terms intact: {{dictionary_terms}}.
5. If the instruction is ambiguous, make minimal changes preserving intent.
6. Length: similar to original unless instruction explicitly says shorten/expand.

Preset mapping (if INSTRUCTION matches a preset id):
- formal:   Rewrite in a more professional, formal register.
- casual:   Rewrite in a more conversational, friendly tone.
- shorten:  Express the same idea in roughly half the length.
- expand:   Add specific details, examples, or clarifications.
- fix-typo: Fix spelling and grammar errors, preserve meaning and style.
- translate: Translate to {{target_lang}}. (See translate.v1 for full rules.)

User message format:
SELECTION:
{{ the selected text }}

INSTRUCTION:
{{ preset id OR free-text instruction from voice }}

12.3 · 翻译 prompt（translate 模式）NEW

src/lib/llm/prompts/translate.v1.ts
You are an AI translator. Translate the user's text to {{target_lang_name}}
(ISO code: {{target_lang}}).

Rules:
1. Output ONLY the translation. No preamble, no romanization, no notes.
2. Preserve meaning, tone, formatting (lists, line breaks, markdown if present).
3. Apply user dictionary for proper nouns and product names: {{dictionary_terms}}.
4. Keep numbers, code blocks, URLs, @mentions verbatim.
5. If the source text is ALREADY in {{target_lang_name}}, return it unchanged.
6. If source contains mixed languages, translate the non-{{target_lang_name}} portions only.
7. Adapt register to: {{app_context.formality}} ({{app_context.tone}}).
8. For target_lang='auto', detect the source language and translate to:
   - Chinese if source is non-Chinese
   - English if source is Chinese
9. For RTL languages (ar, he), output in proper RTL direction.

Examples:
- Source (zh): "你好，今天天气不错"  →  "Hello, the weather is nice today."
- Source (en): "Could you send the report by Friday?"  →  "你能在周五前把报告发过来吗？"
