1. 当前项目定位

项目目标

当前项目是一个 Typeless 风格的网页复刻 demo。它用浏览器麦克风、热键、ASR、LLM 和模拟目标应用输入区，实现“说话即输入”“框选即改写”“语音翻译”和“场景化输出”的完整体验。

项目强调本地可运行、远程可访问、模型路径明确、调用链可验证，而不是只展示静态 UI。

核心输入

左 Alt 按住说话

主输入热键为 Left Alt。按住开始录音，松开后 ASR 转写，再进入 LLM 润色或翻译，最终写入模拟目标应用 textarea 和剪贴板。

选区改写

框选文字后触发 AI 改写

textarea 中框选文字后，页面显示稳定的选区入口与浮动 popover。支持正式化、口语化、缩短、扩写、修错、翻译选区。

语音指令

右 Alt 专用于选区语音改写指令

点击“按语音指令改写”后，页面提示长按 Right Alt 说出改写要求。松开后自动 ASR，使用语音指令改写原选区并替换。

本地模型

Qwen3-ASR 与 Qwen3.5-4B

本地 ASR 和本地 LLM 均通过 Vite dev server 暴露同源 API，避免前端直接读取本地模型目录。页面状态卡显示当前 ASR/LLM 服务、模型名和本地路径。

6. 场景化输出与 Prompt 设计

当前系统不是简单地把 ASR 文本“润色一下”，而是按目标应用上下文生成不同形态的输出。每个场景都有 tone、formality、markdown 和 outputGuidance。

Email

formalplain text

专业邮件格式：必要时包含问候、短段落、明确请求或下一步，以及礼貌结尾。

Slack

team chatcompact

团队聊天格式：直接、紧凑、不使用邮件问候和签名，可用短 bullet。

WeChat

Chinesecasual

自然中文消息：温和、简短、口语化，避免企业邮件式结构。

Code

Markdowndeveloper

开发者场景：保留符号和代码，偏 commit/PR/code-review 语气。

Notes

structuredaction items

结构化笔记：标题、项目符号、决策和行动项，便于扫描。

Casual

friendlynatural

自然表达：保留个人语气，不强行结构化，不添加正式签名。

8. 本地模型调用实现细节

本地 ASR：Qwen3-ASR-1.7B

  • 浏览器上传音频到 /api/asr/qwen3-local/v1/audio/transcriptions。
  • Vite middleware 写入临时文件。
  • Python 脚本使用 ffmpeg 转 WAV：16kHz、单声道。
  • 通过 qwen_asr.Qwen3ASRModel.from_pretrained 加载本地模型。
  • 返回 OpenAI 风格简化响应：{ text, language, duration_ms }。

本地 LLM：Qwen3.5-4B

  • 浏览器调用 /api/llm/qwen35-local/v1/chat/completions。
  • Vite middleware 将请求 JSON 写入临时文件。
  • Python 脚本使用 vLLM 加载 Qwen3.5-4B。
  • 默认注入 no-thinking system instruction。
  • 返回 OpenAI-compatible chat.completion 响应。
// 本地 LLM 端点
POST /api/llm/qwen35-local/v1/chat/completions
{
  "model": "Qwen3.5-4B",
  "messages": [
    { "role": "system", "content": "..." },
    { "role": "user", "content": "..." }
  ],
  "temperature": 0.2
}

10. 状态、存储与错误处理

XState

维护主流程状态：热键按下、ASR 完成、LLM 完成、注入完成、失败状态。状态卡直接显示当前状态或错误信息。

Zustand Persist

保存 app context、模式、target language、endpoint、API key、词典、暗色模式、离线模式等配置。

localforage

历史记录保存在 IndexedDB/localStorage 兼容层中。当前限制 200 条，支持回放和删除。

错误反馈：

ASR、LLM、本地模型脚本、代理请求失败都会进入 toast 和状态机错误上下文。页面不会静默失败，用户能看到具体错误信息。

13. 已知成本与后续建议

当前成本

  • Qwen3.5-4B 本地 LLM 冷启动较慢，首次调用可能需要几十秒到一两分钟。
  • Vite dev middleware 每次请求 spawn Python，适合 demo 验证，不适合生产并发。
  • 本地模型依赖 `.qwen` 环境、vLLM、qwen-asr、ffmpeg 和 CUDA 可用性。
  • 远程访问需要 HTTPS，自签证书需要用户手动允许。

建议演进

  • 将本地 ASR/LLM 从 Vite middleware 独立为常驻 FastAPI 服务，避免重复加载模型。
  • 为本地 LLM 增加加载状态、排队状态和超时重试提示。
  • 为框选改写增加端到端浏览器测试，覆盖远程键盘事件与 textarea 选择。
  • 支持按 endpoint 显示最近一次调用耗时、错误码和模型响应摘要。
  • 将 API key 和模型路径迁移到 `.env.local`，减少硬编码路径。
